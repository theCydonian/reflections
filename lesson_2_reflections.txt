What happens when you initialize a repository? why do you need it?
When you initialize it adds hidden files so it can act like a repository. It initializes with 0 commits. You need to initialize a repository so you can save versions of your project

How is the staging area different from the working directory and the repository? What value do you think it offers?
The staging area is where the file changes go when you use git add before you commit. the working directory is where you originally modify files before you add them to the staging area. When you commit, you move the file from the staging area to the repository
